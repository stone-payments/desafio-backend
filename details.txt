Tecnologias adotadas:
C# - linguagem de programação
Web Api 2 - framework adotado para criação da api rest
Entity framework - framework ORM adotado para interação com banco de dados.
Microsoft Sql server - SGBD adotado para armazenar dados do projeto.
Visual Studio Community 2015 - IDE utilizada para o desenvolvimento.
VSTS + GIT - Ferramentas usadas para criação e gerenciamento de repositório da api.
Postman - Ferramenta usada para fazer chamadas para a API.
IIS - Servidor de aplicações usado localmente pelo visualstudio para testes da api.
Identity Framework - Framework utilizado para criação e gerenciamento de usuários a autenticação dos mesmos.

Instruções básicas para testar a Api:
Para rodar a aplicação deve-se fazer deploy da mesma ou rodar localmente.
Recomenda-se ter instalado o Visual Studio versão 2015 para build da aplicação e debug local.
Caso a escolha seja fazer hospedagem local, o deploy pode ser feito em um Servidor IIS local, usando a versão 10.0 do IIS.
O link abaixo apresenta um simples tutorial para configurar e iniciar a aplicação em um servidor local:
http://dotnetmentors.com/web-api/host-asp-net-web-api-in-iis-using-visual-studio-publish.aspx

Os scripts para criação dos bancos e tabelas devem ser rodados em uma instância do sql server a partir da versão 2014.

Junto ao projeto, encontra-se uma coleção no formato json, que deve ser importada para o Postman.
O Postman pode ser obtido no link: https://www.getpostman.com/
Após instalar o Postman, basta importar a coleção no formato json. Essa coleção possuei chamadas 'HTTP'
com exemplos prontos para testar a api.

Observações: 
- Deve-se criar um usuário para acessar os recursos da api. Sem isso, a api não irá autorizar acesso aos seus recursos.
- A coleção do postman contém um exemplo de como criar o usuário. Após a criação do mesmo, a api enviará os dados registrados. 
- Atente-se para o clientId, pois esse será usado em algumas transações e sem o mesmo, as transações não serão realizadas. É com ele também que é possível buscar
dados no histórico filtrando por cliente.
- Com os dados do usuário, deve-se gerar um token de acesso para usar nas chamadas realizadas para a api.
- Recomenda-se popular as tabelas, seja através da api, seja através de registro direto no banco, para uma melhor experiência na interação com a api.
